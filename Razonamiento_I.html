1. Introducción

Qué problema intenta resolver este capítulo

El capítulo empieza con una observación muy importante: en muchas tareas un agente no posee información completa sobre su entorno. Puede saber algunas cosas, pero otras siguen siendo inciertas. La lógica clásica funciona bien cuando las premisas son seguras, pero no describe de forma natural cuánto se cree en una proposición cuando no hay total certeza.

Para resolver eso, el texto adopta el formalismo probabilístico. La probabilidad permite asignar valores entre 0 y 1 a sucesos o proposiciones y realizar cálculos cuando aparece nueva evidencia. En vez de pensar solo en “verdadero” o “falso”, el capítulo trabaja con grados de certeza.

En el marco del PDF, las probabilidades cumplen el papel de una alternativa probabilística a la representación lógica de hechos inciertos.

Por qué esto es útil

  • Permite representar conocimiento incompleto.
  • Sirve para inferir información nueva a partir de evidencia parcial.
  • Hace posible modelar sensores imperfectos y observaciones dudosas.
  • Ayuda a estructurar el conocimiento para que el razonamiento sea más eficiente.

2. Conceptos fundamentales

Las piezas básicas que usa todo el capítulo

2.1 Variables aleatorias

El texto trabaja con variables aleatorias V1, V2, ..., Vk. Una variable aleatoria representa una característica del dominio sobre la que queremos razonar. En los ejemplos del PDF, esas variables suelen corresponder a proposiciones como si la batería está cargada, si el brazo se mueve o si un bloque se puede levantar.

Cuando una variable representa una proposición, sus valores son Verdadero o Falso. Cuando representa una medida física, sus valores pueden ser numéricos. Cuando representa una categoría, los valores son categóricos. El capítulo remarca que el tipo de valor depende de lo que la variable describa en el dominio.

2.2 Probabilidad conjunta

La probabilidad conjunta habla de varios eventos o variables al mismo tiempo. El texto usa la notación p(V1 = v1, V2 = v2, ..., Vk = vk). Esa expresión indica la probabilidad de que todas las variables tomen simultáneamente esos valores concretos.

El PDF presenta el ejemplo de una moneda para mostrar cómo una función conjunta asigna probabilidades a combinaciones completas de resultados. También recuerda que cada valor debe estar entre 0 y 1, y que la suma total de todas las asignaciones posibles debe dar 1.

2.3 Probabilidad marginal

Una probabilidad marginal se obtiene sumando probabilidades conjuntas más detalladas. Si se quiere saber la probabilidad de una variable sin fijar las demás, se suman todas las combinaciones compatibles con el valor buscado.

El capítulo usa este procedimiento para obtener, por ejemplo, la probabilidad de que la batería esté cargada sumando todas las combinaciones en las que B es verdadero.

2.4 Notación abreviada

El texto también aclara una notación breve para variables proposicionales: cuando una variable aparece afirmada se asume Verdadero, y cuando aparece negada se asume Falso. Esta abreviación solo se usa cuando el contexto deja claro que se habla de una instanciación concreta y no de una distribución completa.

Conjunta Probabilidad de varios hechos al mismo tiempo.
Marginal Probabilidad de una variable aislada a partir de las conjuntas.
Condicional Probabilidad de un hecho sabiendo que otro ya ocurrió.
Evidence Información observada que sirve para inferir lo desconocido.

3. Probabilidad condicional e inferencia

Cómo se actualiza el conocimiento usando evidencia

3.1 Qué es una probabilidad condicional

La probabilidad condicional expresa la probabilidad de una variable dado que otra ya se conoce. El PDF la define como una versión normalizada de la probabilidad conjunta. La idea intuitiva es sencilla: si ya sabemos que cierta evidencia ocurrió, entonces solo nos interesan los casos compatibles con esa evidencia.

El texto ilustra esto con el robot apilador. Por ejemplo, si el brazo no se mueve, el agente quiere estimar la probabilidad de que la batería esté cargada. Ese es precisamente el tipo de pregunta que resuelven las probabilidades condicionales.

3.2 Inferencia probabilística

Inferir significa calcular la probabilidad de una variable desconocida a partir de evidencia conocida. El capítulo presenta esta tarea como la versión probabilística de la inferencia lógica, pero en un contexto donde no se trabaja con verdades absolutas.

El procedimiento general consiste en tomar la evidencia E = e y calcular la probabilidad de una hipótesis V dada esa evidencia. El texto muestra que esto puede hacerse sumando las probabilidades conjuntas compatibles con la observación.

Ejemplo del PDF: a partir de las probabilidades conjuntas de P, Q y R, y usando la evidencia ¬R, se calcula p(Q | ¬R) y el resultado mostrado es 0.75.

3.3 Teorema de Bayes

Uno de los resultados más importantes del capítulo es el teorema de Bayes. El texto lo presenta como una forma de reescribir una probabilidad condicional a partir de la inversa y una normalización. Es clave para el razonamiento diagnóstico, donde se infiere una causa a partir de un efecto observado.

En la práctica, Bayes permite pasar de “sé el efecto” a “qué tan probable es la causa”. Por eso aparece en ejemplos donde la evidencia observada no es una causa directa, sino un síntoma o una señal.

3.4 Qué ocurre con la evidencia incierta

El capítulo aclara que la evidencia usada en los cálculos debe ser cierta o falsa. Si una observación es incierta, se puede modelar añadiendo un nodo hijo que represente la lectura del sensor. Ese nodo sí puede tratarse como evidencia segura, mientras que el nodo original conserva la incertidumbre.

Así, el texto propone representar sensores imperfectos y luego razonar sobre sus lecturas, en vez de asumir que todo lo observado es completamente fiable.

4. Independencia condicional y redes bayesianas

La estructura que hace viable el cálculo

4.1 Independencia condicional

El PDF define independencia condicional como la situación en la que una variable ya no aporta información adicional sobre otra, una vez que se conoce un tercer conjunto de variables. En palabras simples: si ya sé cierto contexto, otra variable deja de ayudarme para predecir el resultado.

El ejemplo del robot lo muestra con claridad: si ya se sabe que la batería está cargada, el indicador no agrega información extra sobre el movimiento del brazo en relación con ese conocimiento.

4.2 Qué es una red bayesiana

Una red bayesiana es un grafo dirigido acíclico cuyos nodos son variables aleatorias. El documento indica que cada nodo es condicionalmente independiente de sus no descendientes, dados sus padres. Esa propiedad no es solo una definición teórica: es la razón por la que la red permite simplificar muchísimo los cálculos.

En lugar de especificar todas las combinaciones posibles de todas las variables, la red permite describir cada nodo mediante su propia tabla de probabilidades condicionales. Así, el conocimiento queda organizado según dependencias locales.

4.3 Interpretación causal

El texto señala que muchas redes bayesianas se construyen interpretando los arcos como relaciones de causa y efecto. Si un experto piensa en causas que producen efectos, suele ser más fácil identificar independencias condicionales razonables y estructurar la red de forma más útil.

4.4 Ventaja principal de la red

La ventaja más importante es la reducción de complejidad. Sin independencias, especificar la distribución conjunta completa puede ser intratable. Con una red bayesiana, el número de valores necesarios baja muchísimo porque cada nodo solo depende de sus padres.

En el ejemplo del capítulo, la red del robot apilador requiere muchas menos probabilidades que una tabla completa de 16 combinaciones.

5. Patrones de inferencia en redes bayesianas

Tres formas típicas de razonar en la red

5.1 Inferencia causal o descendente

Este patrón va de una causa hacia un efecto. El PDF lo usa cuando se calcula la probabilidad de un nodo de consulta a partir de evidencias en sus causas o ancestros. Se trabaja expandiendo la probabilidad del nodo de consulta y sumando sobre los padres que no están observados.

La idea es aprovechar la estructura de la red para no recalcular todo desde cero. Primero se descompone el problema, luego se usan las probabilidades condicionales locales y al final se combinan los resultados.

Resultado mostrado por el texto en el ejemplo causal: p(S | O) = 0.855.

5.2 Inferencia diagnóstica o ascendente

Aquí el razonamiento va desde un efecto observado hacia su causa probable. El texto lo presenta como una aplicación directa del teorema de Bayes. Cuando se observa un síntoma, la pregunta pasa a ser qué causa lo explica mejor.

Este patrón es muy útil cuando la evidencia es una consecuencia de algo no observado. El capítulo remarca que Bayes es el paso principal para transformar el problema en uno causal.

Resultado mostrado por el texto en el ejemplo diagnóstico: p(¬O | ¬S) = 0.88632.

5.3 Inferencia intercausal o justificación

Ocurre cuando una evidencia adicional reduce la plausibilidad de otra causa. Si ya hay una explicación para un efecto observado, una nueva causa puede justificar menos la misma observación. El texto lo muestra con el ejemplo de la batería y el bloque.

En este patrón, la nueva evidencia no simplemente suma información: cambia el peso de otras hipótesis. El PDF destaca que aquí se usa un razonamiento causal incrustado dentro de un razonamiento diagnóstico.

Resultado mostrado por el texto en el ejemplo de justificación: p(¬O | ¬B, ¬S) = 0.030.

6. Separación-d y bloqueos de caminos

Cómo decidir si dos nodos quedan independizados por la evidencia

La separación-d es el mecanismo que permite identificar cuándo dos nodos son condicionalmente independientes dado un conjunto de evidencia. El texto explica que un camino queda bloqueado si se cumple alguna de tres condiciones sobre un nodo intermedio.

  • Si el nodo intermedio pertenece a la evidencia y ambos arcos salen de él.
  • Si el nodo intermedio pertenece a la evidencia y un arco entra y otro sale de él.
  • Si el nodo intermedio no pertenece a la evidencia ni tiene descendientes en ella, y ambos arcos van hacia él.

Si todos los caminos entre dos nodos quedan bloqueados, entonces la evidencia separa-d esos nodos y se puede concluir independencia condicional entre ellos.

Qué aporta esto en la práctica

La separación-d evita recorrer caminos que no afectan a la inferencia. Esto reduce cálculos innecesarios y ayuda a entender por qué ciertas variables sí influyen y otras ya no, dado lo que se conoce.

7. Poliárboles, complejidad y métodos adicionales

Dónde el cálculo se vuelve manejable y qué pasa cuando ya no lo es

7.1 Qué es un poliárbol

Un poliárbol es una red bayesiana donde entre cualquier par de nodos existe un único camino si se ignora la dirección de los arcos. El PDF usa esta clase de redes porque en ellas la inferencia puede hacerse de manera más ordenada.

7.2 Apoyo causal y apoyo evidencial

El capítulo distingue dos situaciones: evidencia por encima del nodo de consulta y evidencia por debajo del nodo de consulta. En el primer caso se usa apoyo causal; en el segundo, apoyo evidencial. Cuando hay evidencia de ambos lados, se combinan ambas formas mediante Bayes y una constante de normalización.

El texto incluso muestra un ejemplo numérico donde se obtiene p(Q | U) = 0.13, después de calcular primero los apoyos causal y evidencial por separado.

7.3 Complejidad

El documento aclara que, aun con estas simplificaciones, la inferencia general puede ser muy costosa. En el peor caso, crece de manera exponencial con el número de variables. Por eso se mencionan estrategias como eliminación del compartimiento, muestreo lógico y agrupamiento.

7.4 Métodos complementarios

  • Muestreo lógico: repite asignaciones aleatorias consistentes con la red para estimar probabilidades.
  • Agrupamiento: combina nodos en “meganodos” para transformar la red en una forma más manejable.
  • Eliminación del compartimiento: evita subcálculos redundantes.
Cuando la red ya no es un poliárbol, los procedimientos recursivos simples pueden dejar de terminar de forma práctica.

8. Ejemplos prácticos del PDF

Resultados que conviene recordar

  • p(Q | ¬R) = 0.75 a partir de una tabla de probabilidades conjuntas de P, Q y R.
  • p(S | O) = 0.855 en una inferencia causal con el robot.
  • p(¬O | ¬S) = 0.88632 en una inferencia diagnóstica.
  • p(¬O | ¬B, ¬S) = 0.030 en una inferencia intercausal.
  • p(Q | U) = 0.13 en el ejemplo numérico del poliárbol abstracto.

Estos ejemplos muestran el mismo patrón: primero se identifica el tipo de evidencia, luego se aplica la estructura de la red y finalmente se normaliza el resultado.

9. Resumen final corto

Lo esencial para memorizar rápido

Razonar con incertidumbre significa trabajar con información incompleta usando probabilidades. El capítulo construye la base con probabilidad conjunta, marginal, condicional, Bayes e inferencia probabilística.

Después muestra que las redes bayesianas y la independencia condicional permiten organizar el conocimiento de forma compacta. La separación-d ayuda a saber qué información queda bloqueada y qué información sigue influyendo.

La idea más importante es esta: la estructura correcta del conocimiento reduce de forma enorme el costo del razonamiento.
