1. La imagen como dato numérico

Una cámara convierte la escena continua del mundo real en una rejilla finita de valores enteros mediante dos pasos:

  • Muestreo: se elige cuántos puntos representar (resolución, ej. 1920×1080).
  • Cuantización: cada intensidad continua se redondea a un nivel entero (rango dinámico).

Representación matricial

Una imagen en grises es una matriz 2D. Cada celda almacena un entero donde 0 = negro y 255 = blanco. Una imagen RGB es un tensor 3D con tres matrices apiladas (R, G, B).

f(x, y) → entero ∈ [0, 255] (8 bits / canal) x = columna, y = fila, valor = intensidad del píxel

Rango dinámico

8 bits — 256 niveles
  • Suficiente para fotografía cotidiana
  • Bajo consumo de memoria
  • Estándar universal (JPEG, PNG)
16 bits — 65 536 niveles
  • Gradientes muy sutiles (medicina, astronomía)
  • Mayor espacio en disco y RAM
  • Edición no destructiva

Histograma e igualización

El histograma cuenta cuántos píxeles tienen cada nivel de intensidad. La igualización redistribuye esos niveles para maximizar contraste usando la función de distribución acumulada (CDF):

g = [ CDF(v) − CDF_min ] / [ N − CDF_min ] × 255 v = valor original del píxel | N = total de píxeles | CDF_min = mínimo no nulo
Aplicación práctica: la igualización revela detalles invisibles en imágenes subexpuestas o con poco contraste, muy útil como preprocesado antes de detectar bordes u objetos.

Operaciones punto a punto

Se aplican píxel a píxel, sin considerar vecinos:

  • Suma de constante: eleva el brillo global (desplaza histograma a la derecha).
  • Resta: baja brillo; comparar dos imágenes detecta cambios.
  • Multiplicación por escalar: amplifica (>1) o comprime (<1) el contraste.

2. Filtrado y detección de bordes

Convolución 2D — el motor del procesamiento espacial

La mayoría de operaciones de filtrado se implementan como una convolución 2D: se desliza un pequeño kernel (matriz de coeficientes) sobre la imagen y se calcula un nuevo valor para cada píxel como suma ponderada de su vecindad.

Kernel / Máscara

Matriz de dimensiones impares (3×3, 5×5…). Sus coeficientes determinan el tipo de efecto: suavizado, realce de bordes, nitidez, etc.

Filtros de paso bajo — suavizado

Atenúan variaciones de alta frecuencia (ruido, detalles finos) y preservan las zonas uniformes.

FiltroKernelCaracterísticaUso típico
PromedioCoeficientes igualesRápido; difumina bordes realesPreprocesado rápido
GaussianoDistribución normal 2DPeso decae desde el centro; preserva mejor la estructuraEliminación de ruido blanco
G(x,y) = 1/(2πσ²) · exp(−(x²+y²)/(2σ²)) σ controla la dispersión del desenfoque

Filtros de paso alto — detección de bordes

Resaltan cambios bruscos de intensidad actuando como derivadas espaciales.

  • Prewitt: calcula diferencias de forma directa con dos kernels (horizontal y vertical).
  • Sobel: añade peso extra en la fila/columna central, lo que reduce sensibilidad al ruido comparado con Prewitt.

Detector de Canny — el estándar de la industria

Propuesto por John F. Canny (1986). No es solo un filtro; es un pipeline algorítmico de cuatro etapas que produce bordes delgados (1 px), con baja tasa de error y buena localización.

  1. Suavizado Gaussiano: elimina ruido antes de calcular gradientes.
  2. Cálculo del gradiente: aplica Sobel para obtener magnitud y dirección.
    Magnitud = √(Gx² + Gy²) Dirección θ = arctan(Gy / Gx)
  3. Supresión de no-máximos: adelgaza bordes comparando cada píxel con sus vecinos en la dirección del gradiente; se conserva solo el máximo local.
  4. Umbralización por histéresis: usa dos umbrales (T_alto, T_bajo).
    • Gradiente > T_alto → borde fuerte (confirmado).
    • Entre T_bajo y T_alto → borde débil; solo se conserva si conecta a un borde fuerte.
    • Por debajo de T_bajo → descartado.
Ventajas de Canny
  • Bordes de 1 px de grosor
  • Bajo número de falsos positivos
  • Configurable con dos umbrales
Limitaciones
  • Más lento que Sobel/Prewitt
  • Sensible a elección de σ
  • Puede fragmentar bordes continuos

3. Puntos clave y descriptores

Un punto clave es una región distintiva de la imagen que puede identificarse y rastrearse aunque cambie la escala, la rotación o la iluminación.

Tipos de puntos clave

TipoDescripciónAlgoritmos
EsquinasCambio brusco de dirección de borde en todas las direccionesHarris, Shi-Tomasi
BlobsRegión con intensidad/color diferente al entorno (manchas, círculos)LoG, DoG, SIFT
Puntos establesReconocibles bajo cambios de escala, rotación e iluminaciónSIFT, SURF, ORB

Harris Corner Detector

Analiza cómo varía la intensidad al desplazar una ventana en distintas direcciones usando la matriz de estructura M:

M = [ Ix² IxIy ] [ IxIy Iy² ] R = det(M) − k·(trace(M))² R grande → esquina | R negativo → borde | R pequeño → zona plana | k ≈ 0.04–0.06
Ventajas
  • Simple y rápido
  • Alta precisión en esquinas
  • Invariante a rotación
Desventajas
  • No maneja bien cambios de escala
  • Sensible a ruido

SIFT — Scale-Invariant Feature Transform

Desarrollado por David Lowe. Detecta y describe puntos clave de forma invariante a escala, rotación y variaciones de iluminación.

  1. Detección de puntos: busca regiones importantes en un espacio escala (pirámide de DoG).
  2. Orientación: asigna una dirección principal al punto basada en el histograma de gradientes locales.
  3. Descriptor: divide la vecindad en subregiones 4×4 y calcula histogramas de gradientes orientados en 8 direcciones → vector de 128 dimensiones.
Concepto clave: el descriptor SIFT de 128 dimensiones actúa como una "huella digital" del punto clave, permitiendo compararlo con puntos de otras imágenes de forma robusta.

Matching de descriptores

Para encontrar el mismo punto en dos imágenes se comparan descriptores con métricas de distancia:

SIFT → Distancia Euclidiana: d = √Σ(xi − yi)² ORB → Distancia de Hamming: cuenta bits diferentes Descriptores con d pequeña son candidatos a corresponder al mismo punto 3D

4. Geometría y múltiples vistas

Modelo de cámara estenopeica (pinhole)

La matriz de cámara P transforma un punto 3D del mundo real en coordenadas de píxeles 2D:

  • Parámetros intrínsecos (K): propiedades internas fijas: distancia focal (fx, fy), punto principal (cx, cy) y factor de inclinación (skew).
  • Parámetros extrínsecos [R|t]: posición y orientación de la cámara en el mundo (rotación R, traslación t).

Calibración de la cámara

Para calibrar se fotografía un tablero de ajedrez desde múltiples ángulos. Cada esquina del tablero proporciona una correspondencia 3D↔2D. El sistema de ecuaciones se resuelve con el método DLT (Direct Linear Transform) y se optimiza con algoritmos no lineales para corregir distorsión radial.

Estereovisión

Dos cámaras separadas por una distancia B (línea base) replican la visión binocular humana para estimar profundidad.

Profundidad (Z) = f · B / d f = distancia focal | B = línea base | d = disparidad (diferencia de posición en px entre imagen izq. y der.)

La geometría epipolar restringe la búsqueda de correspondencias a una línea (la línea epipolar), pasando de buscar en 2D a buscar en 1D, lo que acelera enormemente el proceso.

Triangulación 3D

Dado un par de puntos correspondientes en dos imágenes, se trazan rayos desde cada centro óptico. En teoría se intersectan en el punto 3D real. En la práctica el ruido impide la intersección exacta, por lo que se usa mínimos cuadrados para encontrar el punto que minimiza el error de reproyección. El resultado es una nube de puntos 3D de la escena.

5. Pipeline clásico y clasificadores

Estructura del pipeline de visión

El pipeline clásico procesa la imagen de forma secuencial hasta producir una decisión:

  1. Captura: la cámara obtiene la imagen; se corrige iluminación si es necesario.
  2. Preprocesado: filtros para reducir ruido y mejorar contraste.
  3. Extracción de características: bordes, esquinas, texturas, formas → vector numérico.
  4. Clasificación / estimación: SVM, k-NN u otro clasificador decide la categoría.
  5. Acción: el sistema ejecuta la decisión (ej. el robot frena ante un obstáculo).

SVM — Máquina de Vectores Soporte

Busca el hiperplano de máximo margen que separa las clases de datos. Los puntos más cercanos a esa frontera se llaman vectores soporte.

Ventajas
  • Funciona bien con datos de alta dimensión
  • Robusto frente al sobreajuste
  • Kernels para separaciones no lineales
Desventajas
  • Lento con grandes conjuntos de datos
  • Sensible a la elección del kernel y parámetros
  • Poca interpretabilidad

k-NN — k vecinos más cercanos

Clasifica un nuevo ejemplo buscando los k ejemplos más similares en el conjunto de entrenamiento y haciendo votación por mayoría.

Idea clave: no hay entrenamiento explícito; todo el trabajo ocurre en la consulta. Sencillo de implementar, pero lento con bases de datos grandes porque debe calcular la distancia a todos los ejemplos almacenados.

RANSAC — eliminación de outliers

Algoritmo robusto para ajustar modelos cuando los datos contienen errores o coincidencias incorrectas (outliers). Funciona iterando:

  1. Seleccionar aleatoriamente un subconjunto mínimo de puntos.
  2. Calcular un modelo candidato con esos puntos.
  3. Contar cuántos puntos totales son consistentes con el modelo (inliers).
  4. Conservar el modelo con más inliers tras N iteraciones.

Validación y sobreajuste

El conjunto de datos se divide en entrenamiento (aprender patrones) y prueba (evaluar generalización). El sobreajuste ocurre cuando el modelo memoriza el entrenamiento pero falla con datos nuevos; se mitiga con más datos, regularización o ajuste fino de parámetros.
