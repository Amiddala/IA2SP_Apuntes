1. Análisis de sentimiento — ejemplo completo

El análisis de sentimiento es una tarea de PLN que determina la opinión o emoción expresada en un texto. Es uno de los mejores ejemplos para ilustrar el flujo completo de un sistema de PLN, desde el texto crudo hasta una clasificación útil.

Objetivo del sistema

Dado un comentario de texto, clasificarlo en: Positivo | Negativo | Neutro. Por ejemplo: "la plataforma funciona muy bien y es fácil de usar" → Positivo.

El pipeline completo

📝 Texto crudo
→
🔧 Preprocesado
→
🔢 Representación
→
🤖 Modelo
→
✅ Clasificación

Cada etapa transforma el texto en una representación más abstracta que el modelo puede procesar. Un error en etapas tempranas degrada el rendimiento en todas las siguientes.

2. Preprocesado y representación

Tokenización y limpieza

La tokenización divide la oración en unidades (tokens). Además se normalizan caracteres y se eliminan elementos irrelevantes:

"la clase fue EXCELENTE!!!" ↓ tokenizar + normalizar ["la", "clase", "fue", "excelente"] Se eliminan mayúsculas, puntuación excesiva y caracteres especiales
Importancia del preprocesado: un modelo entrenado con texto ruidoso aprende patrones incorrectos. Limpiar los datos es tan importante como la arquitectura del modelo.

Representación vectorial mediante embeddings

Las computadoras no entienden palabras directamente; necesitan números. Cada token se convierte en un vector numérico donde palabras semánticamente similares tienen vectores próximos:

"excelente" → (0.82, 0.11, 0.65, …) "muy bueno" → (0.79, 0.14, 0.63, …) ← vectores cercanos "terrible" → (0.05, 0.91, 0.12, …) ← vector lejano La distancia entre vectores captura similitud semántica

3. Modelo de clasificación y evaluación

Función Softmax para clasificación multiclase

Una vez representado el texto como vector z_d, el clasificador calcula la probabilidad de cada clase con la función softmax:

P(y = k | d) = exp(wₖᵀ · z_d) / Σⱼ exp(wⱼᵀ · z_d) wₖ = parámetros de la clase k | z_d = representación del documento

Ejemplo de salida del clasificador

Para el texto "el servicio fue lento y frustrante":

ClaseProbabilidadDecisión
Positivo0.05
Neutro0.10
Negativo0.85← Clase elegida

Comprensión de contexto

Los modelos modernos no solo miran palabras aisladas; entienden que el sentimiento global puede contradecir palabras individuales positivas:

Ejemplo de contexto: "la clase parecía buena pero el examen fue terrible" → el modelo debe entender que "terrible" domina el sentimiento global sobre "buena".

Métricas de evaluación

Tras entrenar el modelo se mide su calidad con datos de prueba nunca vistos:

TP/(TP+FP)
Precisión
TP/(TP+FN)
Recall
2·P·R/(P+R)
F1-Score
  • Precisión: de todos los positivos predichos, ¿cuántos eran realmente positivos?
  • Recall: de todos los positivos reales, ¿cuántos detectó el modelo?
  • F1-Score: media armónica de precisión y recall; equilibra ambas métricas.

Flujo operativo en producción

  1. Entrenamiento: el modelo aprende patrones con datos etiquetados.
  2. Validación: se ajustan hiperparámetros con datos de validación separados.
  3. Despliegue: el modelo analiza nuevos comentarios en tiempo real.
  4. Monitoreo y re-entrenamiento: el rendimiento se vigila; si cae, se re-entrena con datos nuevos.

5. Nexos entre Visión y PLN

Visión por Computadora

Convolución 2D (Filtros)

El kernel se desliza sobre la imagen calculando sumas ponderadas locales para detectar patrones espaciales (bordes, texturas).

Procesamiento de Lenguaje Natural

Mecanismo de Atención

Para cada token, calcula sumas ponderadas sobre toda la secuencia para capturar dependencias semánticas arbitrariamente lejanas.

Ambos son mecanismos de agregación ponderada, pero con alcance diferente: la convolución es local (vecindad del píxel), mientras que la atención es global (toda la secuencia).

6. Conclusiones

Lecciones fundamentales de Visión por Computadora

  • Una imagen es una matriz de números; entender su representación matemática es el punto de partida.
  • Los filtros (Gaussiano, Sobel, Canny) son operaciones matemáticas simples pero poderosas para extraer información.
  • Los descriptores (SIFT, ORB) resuelven el problema de comparar regiones entre imágenes distintas con invarianza a transformaciones.
  • La calibración y la geometría epipolar son puentes entre las coordenadas 2D de la imagen y el mundo 3D real.
  • Los clasificadores clásicos (SVM, k-NN) funcionan bien cuando las características son bien elegidas; las CNNs las aprenden automáticamente.

Lecciones fundamentales de PLN

  • El lenguaje es ambiguo, variable y dependiente del contexto; no hay solución perfecta, solo mejores aproximaciones.
  • La representación del texto es crucial: BoW ignora semántica; los embeddings la capturan; los modelos contextuales la maximizan.
  • La progresión Naive Bayes → HMM → CRF refleja la relajación progresiva de supuestos de independencia.
  • LSTM y GRU resuelven el olvido de las RNNs, pero siguen siendo secuenciales.
  • Los Transformers paralelizaron el procesamiento y desbloquearon los LLMs modernos.

El paradigma que domina hoy

Métodos modernos (Deep Learning)
  • Aprenden representaciones automáticamente
  • Estado del arte en casi todas las tareas
  • Un solo modelo para muchas tareas (LLMs)
  • Mejoran con más datos y cómputo
Métodos clásicos (aún relevantes)
  • Interpretables y auditables
  • Funcionan con pocos datos
  • Bajo costo computacional
  • Necesarios en sistemas embebidos y robótica
Reflexión final: comprender los métodos clásicos no es obsoleto. Son la base conceptual sobre la que se construyeron las arquitecturas modernas, y siguen siendo la herramienta correcta cuando los recursos son limitados o la interpretabilidad es un requisito no negociable.
